Development Environment:

I did this project on Ubuntu 24 running inside VMware on my personal laptop. Setting up the environment took some time because I had to install many tools I had not used before.

Tools I Used:
Renode
I used Renode version 1.16 to emulate the SiFive FE310 RISC-V microcontroller. I had never used Renode before this project. I learned how to write .repl platform description files and .resc scripts to load and run firmware.

RISC-V
This is the cross compiler I used to compile C code for RISC-V. I installed it from ubuntu package manager. The hardest part was figuring out the correct flags for rv32imac architecture and linking with picolibc.

Picolibc

I needed a C standard library for bare metal RISC-V. I found that picolibc was already available as a package for riscv64-unknown-elf on Ubuntu. I had to manually point the compiler to the include and library paths.

Wasm3

This is the WebAssembly runtime I embedded into the firmware. I cloned it from GitHub and compiled it as part of the firmware. Learning how to use the Wasm3 API to load and call WASM functions took some time reading the documentation and examples.

ChatGPT (chat.openai.com) 

used for code suggestions and debugging help

I used ChatGPT mainly as a search engine replacement. When I got an error I did not understand I searched for it and also asked ChatGPT to explain what it means. I also used it to understand concepts like how WebAssembly imports and exports work and what a WASM runtime is.

 For Debugging
When I got errors I searched them online and sometimes asked AI to help explain. For example:

- type sig index out of bounds  - searched this error, found it relates to WASM section parsing
- undefined reference to _exit - learned about bare metal stubs
- memory allocation failed - learned about heap size configuration in Wasm3
- BSS not being zeroed - learned about startup code and crt0
 For Code
